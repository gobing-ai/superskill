# A: Portable plugin scripts via install-time staging

## Goal
Make plugin-level `scripts/` a first-class install artifact with a documented, portable invocation contract so any plugin can ship skill-related scripts without one-off absorption into the CLI.
## Scope
- **In:** Install-time staging of `plugins/<plugin>/scripts/**`; `superskill script path` helper; dual contract (path standard + optional `script run` registry); guide rewrite under `docs/help/`; non-hook `validate-response` migration; hook-path unification design (R6-B); ADR/doc supersession for ADR-015 / ADR-022 / `04_DESIGN` / `bundled_plugin`.
- **Out:** Promoting scripts to a workspace package; runtime discovery of arbitrary third-party scripts without install; requiring Bun on all targets; deleting hook emitters before a replacement design lands.
